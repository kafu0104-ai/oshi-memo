# 🤍 ARCHITECTURE

# システム構成

推しメモは「イベント」を中心に情報を管理します。

すべてのデータはイベントに紐付き、
画面表示や通知もイベントを起点として処理されます。

---

# システム構成

```
App
│
├── Home
├── Events
│    ├── Goods
│    ├── Ticket
│    ├── Transaction
│    ├── Todo
│    └── Notification
│
└── Settings
```

---

# データ構造

```
Event
├── Goods
├── Ticket
├── Transaction
├── Todo
└── Notification
```

Eventを親データとし、
各機能は独立したデータとして管理します。

---

# 画面構成

Home

↓

Event一覧

↓

Event詳細

↓

各機能

- Goods
- Ticket
- Transaction
- Todo

必要に応じて画面を切り替えながら利用します。

---

# ホーム画面

ホーム画面は
「ダッシュボード」として機能します。

表示内容

- やることリスト
- 通知
- 今後のイベント
- 未精算件数

詳細な操作は各画面で行います。

---

# イベント画面

イベント画面では、

1つのイベントに関する情報をまとめて管理します。

イベント詳細から

- グッズ
- チケット
- 精算
- やること

へアクセスできます。

---

# データ更新フロー

```
Goods 更新
      │
      ▼
Todo 更新
      │
      ▼
Notification 更新
      │
      ▼
Home 更新
```

同様に、

```
Ticket 更新
```

```
Transaction 更新
```

についても
Homeまで自動更新されます。

---

# 通知フロー

```
イベント
      │
      ▼
期限設定
      │
      ▼
通知生成
      │
      ▼
ホーム表示
```

完了した場合

```
完了
      │
      ▼
通知停止
      │
      ▼
ホーム更新
```

となります。

---

# やること生成フロー

Todoは

- Goods
- Ticket
- Transaction

の状態から自動生成される場合があります。

生成されたTodoも、
ユーザーが自由に編集できます。

---

# モジュール構成

```
App
│
├── Home

├── Goods
├── Ticket
├── Transaction
├── Todo
├── Notification
└── Settings
```

各モジュールは独立しており、
必要なデータのみを参照します。

---

# 設計方針

推しメモでは

**「疎結合な設計」**

を採用します。

各機能を独立させることで、

- 保守しやすい
- 拡張しやすい
- 再利用しやすい

構造を目指します。
