# 推しメモについて

## コンセプト

> 推し活で「あ、忘れてた…」をなくすアプリ。

推しメモは、推し活をもっと快適にするためのイベント管理アプリです。

グッズ購入、チケット、立替精算、受け渡しなど、
推し活では多くの情報を同時に管理する必要があります。

しかし現在は、

- メモアプリ
- スプレッドシート
- LINE
- 電卓
- カレンダー

など複数のアプリを行き来しながら管理することが多く、
情報が分散しやすいという課題があります。

推しメモは、それらを一つのアプリにまとめることで、

**「推し活に集中できる環境」**

を提供します。

---

# 解決したい課題

推し活では、次のような「忘れた」が起こりやすくなります。

- グッズの買い忘れ
- 売り切れ後の再入荷確認忘れ
- 事後通販の購入忘れ
- チケット当落確認忘れ
- 支払い期限忘れ
- 発券忘れ
- 特典配分ミス
- 立替精算漏れ
- グッズの受け渡し忘れ

これらはどれも小さなことですが、
積み重なると推し活の満足度を大きく下げてしまいます。

---

# プロダクトの考え方

推しメモは

**「商品を管理するアプリ」**

ではありません。

目指しているのは、

**「次に自分がやるべきことを忘れないアプリ」**

です。

例えば、

- 今日は当落発表の日
- 支払い期限まであと2日
- 今日から発券開始
- 再入荷待ちの商品がある
- Aさんへの受け渡しがまだ終わっていない

など、

"今やるべきこと"

が自然に分かる設計を目指しています。

---

# ターゲットユーザー

推しメモは次のようなユーザーを想定しています。

- グッズを複数購入する人
- ライブやイベントによく参加する人
- 友人と共同購入する人
- 特典交換をする人
- チケット管理をまとめたい人
- 推し活をもっと効率よく楽しみたい人

---

# プロダクトの価値

推しメモが提供する価値は

**「管理」ではありません。**

提供したい価値は、

**安心して推し活を楽しめること。**

そのために、

- 忘れない
- 迷わない
- 探さない

という3つを大切にしています。

---

# 今後の展望

今後は

- Push通知
- クラウド同期
- 複数端末対応
- PWA対応

などを通じて、

どこでも安心して利用できる推し活アシスタントを目指します。

---

# 開発ポリシー

新しい機能を増やすことよりも、

**「本当に推し活で役立つか」**

を最優先に考えます。

ユーザー目線を大切にし、
実際の推し活で得られた気づきを反映しながら、
継続的に改善を続けていきます。

---

# 🤍 Vision

推し活は、

「管理すること」が目的ではありません。

好きな人や作品を、
もっと楽しく応援することが目的です。

推しメモは、
その時間を少しでも増やすためのアシスタントであり続けます。
