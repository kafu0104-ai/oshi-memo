# 🤍 SPEC

# 機能仕様

推しメモで提供する機能と、その動作仕様を定義します。

---

# イベント管理

イベントを起点として、すべての情報を管理します。

イベントには以下の情報を紐付けることができます。

- グッズ
- チケット
- 精算
- やること
- 通知

イベントを削除した場合は、関連するデータも削除されます。

---

# グッズ管理

グッズには購入状況を設定できます。

状態

- 未購入
- 保留
- 購入済み
- 売切れ
- 再入荷待ち
- 事後通販待ち
- 見送り

状態はユーザーが変更できます。

「売切れ」は販売状況を表し、
その後に

- 再入荷待ち
- 事後通販待ち
- 見送り

のいずれかへ変更できます。

---

# 特典計算

購入金額に応じて特典枚数を計算します。

イベントごとに

- 対象金額
- 特典内容

を設定できます。

特典枚数は自動計算されます。

---

# チケット管理

チケットでは

- 応募
- 当落
- 支払い
- 発券
- 入場

までを管理します。

必要に応じて

- 同行者
- 座席
- メモ

も登録できます。

---

# 精算管理

立替・共同購入を管理します。

管理内容

- 誰が支払うか
- 誰に渡すか
- 精算状況
- 支払い方法

精算内容は共有用にまとめて表示できます。

---

# やることリスト

やることリストには、

ユーザーが行動する必要がある項目のみ表示します。

例

- 支払い
- 発券
- 受け渡し
- 再入荷確認
- 事後通販開始

完了した項目は一覧から除外されます。

---

# 通知

通知は

「ユーザー自身が対応する必要があるもの」

のみ作成されます。

例

- 当落発表
- 支払い期限
- 発券開始
- イベント前日
- 再入荷予定日
- 事後通販開始
- 事後通販終了

通知タイミング

- 当日
- 1日前
- 2日前
- 3日前
- 1週間前
- カスタム

完了した通知は自動的に停止します。

---

# ホーム画面

ホーム画面では、

現在対応が必要な情報を優先して表示します。

表示例

- やること件数
- 未精算件数
- 今後のイベント
- 通知

詳細は各画面から確認できます。

---

# データ更新

情報を更新した場合は、
関連するデータも自動更新されます。

例

- 支払い完了
　→ やることリスト更新
　→ 通知停止

- 発券完了
　→ やることリスト更新
　→ 通知停止

- 精算完了
　→ 未精算件数更新

---

# 共有

共有機能では

- 精算内容
- 購入内容

などを、メッセージアプリへ共有できる形式で出力します。

共有内容は必要に応じて編集できます。

---

# エラーハンドリング

入力内容に不備がある場合は、
分かりやすいメッセージを表示します。

削除などの重要な操作では、
確認ダイアログを表示します。

# 基本方針

推しメモは、

**「ユーザーが今やるべきことを分かりやすくする」**

ことを最優先とします。

情報を管理することではなく、

**行動につながる情報を提供すること**

を目的として設計します。
