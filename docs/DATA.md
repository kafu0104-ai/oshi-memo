# 🤍 DATA

# データモデル

推しメモは、イベントを中心に各データを管理します。

```
Event
 ├─ Goods
 ├─ Ticket
 ├─ Transaction
 ├─ Todo
 └─ Notification
```

それぞれのデータは独立していますが、
すべてイベントに紐付いて管理されます。

---

# Event

イベント情報を管理します。

例

- ライブ
- 展示会
- コラボカフェ
- ポップアップショップ
- オンライン販売

主な情報

- イベント名
- 開催日
- 会場
- メモ

---

# Goods

グッズ情報を管理します。

主な情報

- 商品名
- カテゴリ
- 金額
- 数量
- 購入者
- 状態
- 特典情報
- メモ

状態例

- 未購入
- 保留
- 購入済み
- 売切れ
- 再入荷待ち
- 事後通販待ち
- 見送り

---

# Ticket

チケット情報を管理します。

主な情報

- 公演名
- 枚数
- 当落
- 支払い状況
- 発券状況
- 座席
- 同行者
- メモ

---

# Transaction

立替・精算情報を管理します。

主な情報

- 相手
- 金額
- 支払い担当
- 精算状況
- 支払い方法
- メモ

精算状況例

- 未精算
- 精算予定
- 精算済み

---

# Todo

やることを管理します。

主な情報

- タイトル
- 内容
- 期限
- 優先度
- 完了状態

Todoは必要に応じて、
各データから自動生成される場合があります。

---

# Notification

通知情報を管理します。

主な情報

- 通知内容
- 通知日時
- 通知対象
- 通知タイミング
- 完了状態

通知は、
ユーザーが行動する必要があるもののみ作成されます。

---

# データの関係

```
Event
├─ Goods
├─ Ticket
├─ Transaction
├─ Todo
└─ Notification
```

各データはイベント単位で管理されます。

TodoやNotificationは、
Goods・Ticket・Transactionの状態に応じて生成・更新されます。

---

# データ設計ポリシー

推しメモでは、

**「必要な情報を必要な場所にだけ持つ」**

ことを基本方針とします。

同じ情報を複数の場所で保持せず、
データの重複をできるだけ避けることで、
管理しやすく、一貫性のある構造を目指します。
